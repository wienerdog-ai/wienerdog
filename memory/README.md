# Dogfood vault

This is the Wienerdog repo's own memory — the first Wienerdog deployment, hand-rolled before the product exists, then extracted into `templates/vault/` as it stabilizes. Rule: **if the team routes around a Wienerdog feature during development, that's a P1 product bug.**

- `lessons/inbox.md` — append-only; every session adds one bullet per lesson/gotcha, prefixed with its WP id. Don't organize it; the dream job consolidates.
- `lessons/consolidated/` — dream output: deduped, promoted lessons.
- `research/` — wd-researcher memos (`YYYY-MM-DD-topic.md`).
- `daily/` — daily development logs.
- `transcripts/` — session capture queue (gitignored; stays local).
